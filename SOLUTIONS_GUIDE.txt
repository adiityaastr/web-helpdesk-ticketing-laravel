## ISSUE #1: Missing Authorization di KnowledgeBaseController

### Current Code (VULNERABLE):
`php
// app/Http/Controllers/Admin/KnowledgeBaseController.php
public function destroy(KnowledgeBase ): RedirectResponse
{
    ->delete();
    return redirect()->route('admin.knowledge-base.index')->with('success', 'Artikel berhasil dihapus.');
}
`

### Problem:
- Tidak ada authorization check
- Setiap staff bisa delete artikel milik staff lain
- Tidak ada audit trail

### Solution:

#### Step 1: Create KnowledgeBasePolicy
`php
// app/Policies/KnowledgeBasePolicy.php
<?php
namespace App\Policies;

use App\Models\KnowledgeBase;
use App\Models\User;

class KnowledgeBasePolicy
{
    public function viewAny(User ): bool
    {
        return ->isStaff();
    }

    public function view(User , KnowledgeBase ): bool
    {
        return ->isStaff();
    }

    public function create(User ): bool
    {
        return ->isStaff();
    }

    public function update(User , KnowledgeBase ): bool
    {
        return ->isStaff() && (->id === ->author_id || ->hasRole('admin'));
    }

    public function delete(User , KnowledgeBase ): bool
    {
        return ->isStaff() && (->id === ->author_id || ->hasRole('admin'));
    }
}
`

#### Step 2: Register Policy
`php
// app/Providers/AuthServiceProvider.php
protected \ = [
    KnowledgeBase::class => KnowledgeBasePolicy::class,
];
`

#### Step 3: Update Controller
`php
public function destroy(KnowledgeBase ): RedirectResponse
{
    \->authorize('delete', \);
    \->delete();
    return redirect()->route('admin.knowledge-base.index')
        ->with('success', 'Artikel berhasil dihapus.');
}
`

## ISSUE #3: Inconsistent Cache Strategy

### Current Code (PROBLEMATIC):
`php
// Multiple locations with hardcoded cache keys
Cache::forget('admin_dashboard_stats');
Cache::forget('admin_dashboard_charts');
Cache::forget("portal_dashboard_stats_{->id}");
`

### Problem:
- Cache keys hardcoded di multiple places
- Mudah lupa invalidate cache
- Sulit maintain dan refactor

### Solution:

#### Step 1: Create Cache Manager
`php
// app/Services/CacheManager.php
<?php
namespace App\Services;

use Illuminate\Support\Facades\Cache;

class CacheManager
{
    const ADMIN_DASHBOARD_STATS = 'admin_dashboard_stats';
    const ADMIN_DASHBOARD_CHARTS = 'admin_dashboard_charts';
    const PORTAL_DASHBOARD_STATS = 'portal_dashboard_stats';
    const ADMIN_SAW_SCORES = 'admin_saw_scores';
    const REFERENCE_CATEGORIES = 'reference_categories';
    const REFERENCE_DEPARTMENTS = 'reference_departments';
    const USER_AUTH = 'user_auth';
    const USER_NOTIF_COUNT = 'user_notif_count';
    
    const TTL_SHORT = 300;      // 5 minutes
    const TTL_MEDIUM = 3600;    // 1 hour
    const TTL_LONG = 86400;     // 24 hours

    public static function forgetAdminDashboard(): void
    {
        Cache::tags(['admin_dashboard'])->flush();
    }

    public static function forgetPortalDashboard(int \): void
    {
        Cache::tags(["portal_dashboard_{}"])->flush();
    }

    public static function forgetReferences(): void
    {
        Cache::tags(['references'])->flush();
    }

    public static function forgetUserAuth(int \): void
    {
        Cache::forget(self::USER_AUTH . "_{}");
    }

    public static function forgetUserNotifications(int \): void
    {
        Cache::forget(self::USER_NOTIF_COUNT . "_{}");
    }
}
`

#### Step 2: Update Services
`php
// app/Services/TicketService.php
public function createTicket(Request \, User \): Ticket
{
    // ... create ticket logic
    
    // Use CacheManager instead of hardcoded keys
    CacheManager::forgetAdminDashboard();
    CacheManager::forgetPortalDashboard(\->id);
    
    return \;
}
`

#### Step 3: Use Tags in Cache Calls
`php
// app/Services/DashboardService.php
public function getAdminStats(): array
{
    return Cache::tags(['admin_dashboard'])->remember(
        'admin_stats',
        CacheManager::TTL_MEDIUM,
        function () {
            // ... stats calculation
        }
    );
}
`

// app/Exceptions/TicketException.php
<?php
namespace App\Exceptions;

use Exception;

class TicketException extends Exception
{
    public static function notResolved(): self
    {
        return new self('Ticket is not in resolved status', 422);
    }

    public static function alreadyConfirmed(): self
    {
        return new self('Resolution has already been confirmed', 422);
    }

    public static function unauthorized(): self
    {
        return new self('Only ticket reporter can perform this action', 403);
    }

    public static function cannotRate(): self
    {
        return new self('Ticket must be resolved to rate', 422);
    }

    public static function alreadyRated(): self
    {
        return new self('Rating has already been provided', 422);
    }
}
`

    }

    if (\->resolved_confirmed_at) {
        throw TicketException::alreadyConfirmed();
    }

    // ... rest of logic
}
`

#### Step 3: Handle in Controller
`php

## ISSUE #8: Missing Ticket Status State Machine

### Current Code (NO VALIDATION):
`php
// app/Http/Requests/UpdateTicketRequest.php
'status' => ['required', Rule::in(['in_progress', 'resolved', 'closed', 'cancelled'])],
`

### Problem:
- Ticket bisa jump dari 'open' ke 'closed' tanpa 'resolved'
- Tidak ada business logic validation
- Inconsistent state transitions

### Solution:

#### Step 1: Create Status Enum
`php
// app/Enums/TicketStatus.php
<?php
namespace App\Enums;

enum TicketStatus: string
{
    case OPEN = 'open';
    case IN_PROGRESS = 'in_progress';
    case RESOLVED = 'resolved';
    case CLOSED = 'closed';
    case CANCELLED = 'cancelled';

    public function canTransitionTo(self \): bool
    {
        return match (\) {
            self::OPEN => in_array(\, [
                self::IN_PROGRESS,
                self::CANCELLED,
            ]),
            self::IN_PROGRESS => in_array(\, [
                self::RESOLVED,
                self::CANCELLED,
            ]),
            self::RESOLVED => in_array(\, [
                self::CLOSED,
                self::IN_PROGRESS,
            ]),
            self::CLOSED => false,
            self::CANCELLED => false,
        };
    }
}
`

#### Step 2: Update Service
`php
// app/Services/TicketService.php
public function updateTicket(Ticket \, array \, User \): Ticket
{
    if (isset(\['status'])) {
        \ = TicketStatus::from(\->status);
        \ = TicketStatus::from(\['status']);

        if (!\->canTransitionTo(\)) {
            throw new \InvalidArgumentException(
                "Cannot transition from {->value} to {->value}"
            );
        }
    }

    // ... rest of logic
}
`
